Danke für dein Interesse, die Inhalte zu verbessern! Korrekturen, Ergänzungen und Hinweise sind ausdrücklich willkommen.

## So funktioniert es

**Sei mutig** — wie bei Wikipedia: Bearbeite einfach direkt die betroffene Datei auf GitHub („Edit"-Stift oben rechts), egal ob Tippfehler, kaputter Link, ein neuer Abschnitt oder gleich ein **ganzer Gastartikel** (neue Datei unter [`blog/`](blog/) anlegen, Frontmatter von einem bestehenden Artikel übernehmen). GitHub erstellt daraus automatisch einen Pull Request, der von mir in den darauffolgenden Tagen geprüft und freigegeben wird.

Wer lieber zuerst diskutieren möchte oder eine Frage hat, ohne schon einen Textvorschlag zu haben, kann alternativ ein [Issue](../../issues/new/choose) eröffnen — nötig ist das aber nicht.

## Wichtig zu wissen

Dieses Repository ist die Markdown-Ausgabe der Inhalte von [rafaelpfister.ch](https://rafaelpfister.ch) — die Website ist die primäre Quelle. Angenommene Änderungen übernehme ich deshalb zuerst dort; das Repository wird anschliessend synchronisiert. Dein Pull Request wird dann ggf. nicht klassisch gemerged, sondern mit Verweis auf den Sync-Commit als **übernommen** geschlossen — dein Beitrag ist damit auf der Website und hier im Repo enthalten.

Jeder Vorschlag wird von mir persönlich geprüft. Es kann daher ein paar Tage dauern, bis du eine Rückmeldung bekommst.
